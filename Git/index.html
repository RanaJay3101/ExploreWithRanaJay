Git and GitHub Command Guide

Git Configuration Commands

git config --global user.name "name"

Sets the username for all repositories on your system.

git config --global user.email "email"

Sets the email address for all repositories.

git config --global core.editor "code --wait"

Configures Visual Studio Code as the default editor.

git config --global core.autocrlf "input"

Converts line endings for compatibility between Unix and Windows systems.

git config --global -e

Opens the global configuration file in your editor.

git config --list

Displays all the configuration settings.

Initializing and Cloning Repositories

git init

Creates a new Git repository in the current directory.

git clone <repo-url>

Copies an existing repository to your local machine.

Options:

  • --depth 1: Clone with limited history (only the latest commit).
  • --branch <branch-name>: Clone a specific branch.

Working with the Staging Area

git status

Shows the status of the working directory and staging area.

git status -s

Provides a simplified view of the status.

git add <file-name>

Moves changes to the staging area for the specified file.

git add .

Stages all changed files in the working directory.

git reset <file-name>

Removes a file from the staging area but keeps the changes in the working directory.

Committing Changes

git commit -m "Commit message"

Saves the staged changes with a descriptive message.

git commit --amend -m "New commit message"

Modifies the previous commit with a new message or additional changes.

git commit -a -m "Commit message"

Stages and commits all tracked files in one step.

Viewing Commit History

git log

Displays a detailed history of commits.

git log --oneline

Shows each commit as a single line.

git log --oneline --graph

Displays the commit history as a graph.

git log --author="Author Name"

Filters the commit history to show only commits by a specific author.

git log -p

Shows the differences introduced in each commit.

Working with Branches

git branch

Lists all branches in the repository.

git branch <new-branch-name>

Creates a new branch without switching to it.

git switch <branch-name>

Switches to the specified branch.

git switch -c <new-branch-name>

Creates a new branch and switches to it.

git branch -m <new-branch-name>

Renames the current branch.

git branch -d <branch-name>

Deletes a branch that has been merged.

git branch -D <branch-name>

Deletes a branch regardless of its merge status.

Merging Branches

git merge <branch-name>

Combines the specified branch into the current branch.

git merge --ff-only <branch-name>

Moves the current branch pointer forward without creating a new commit.

git merge --squash <branch-name>

Combines all changes from the branch into a single commit.

git merge --abort

Stops the merge process and reverts to the previous state.

Handling Merge Conflicts

git status

Lists all files with conflicts.

Options to Resolve Conflicts:

  • Accept Current Change: Keeps the changes from the current branch.
  • Accept Incoming Change: Accepts the changes from the branch being merged.
  • Accept Both Changes: Combines changes from both branches.

git add <resolved-file>

Stages the resolved file.

git commit -m "Resolved merge conflict"

Finalizes the merge with a commit.

Undoing Changes

git reset <file-name>

Removes a file from the staging area.

git checkout -- <file-name>

Reverts a file back to its last committed state.

git reset --hard <commit-hash>

Resets the working directory and staging area to match a specific commit, discarding all changes.

git reset --soft <commit-hash>

Moves the HEAD to a previous commit but keeps changes in the staging area.

git revert <commit-hash>

Creates a new commit that undoes the changes from a specific commit.

Stashing Changes

git stash

Saves changes to a temporary area, allowing you to switch branches without committing.

git stash apply

Restores the stashed changes.

git stash drop

Deletes the most recent stash.

git stash clear

Removes all stashed changes.

Collaborating with GitHub

git remote add origin <repo-url>

Links your local repository to a remote repository.

git remote -v

Displays the remote repositories linked to your local repository.

git push -u origin <branch-name>

Uploads your branch to the remote repository.

git pull

Fetches and merges changes from the remote repository into the current branch.

git fetch

Downloads changes from the remote repository without merging.

git push origin --delete <branch-name>

Removes a branch from the remote repository.

Common GitHub Workflow

  1. Main person creates the repository and uploads initial files to GitHub.
  2. Collaborators clone the repository to their local machine.
  3. Collaborators create their own branches and make changes.
  4. Collaborators push their branches to GitHub and inform the team.
  5. The project maintainer fetches the new branches and merges them into the main branch.
